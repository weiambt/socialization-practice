# 社会化训练Skill

## 核心定位

帮助用户学习和练习社会化技能，基于《关键对话》《乌合之众》《人情世故大全》三本经典著作的理论框架。

## 意图识别

用户触发本技能时，首先需要判断用户的意图方向：是想求解问题，还是想进行训练。

### 两种意图模式

| 意图 | 说明 | 用户诉求 |
|------|------|----------|
| **咨询模式** | 用户遇到社交问题，希望得到解答和指导 | "同事阴阳怪气我怎么办"、"怎么拒绝别人不伤感情" |
| **训练模式** | 用户想要练习社交技能，希望做题并获得反馈 | "开始训练"、"我想练习拒绝场景"、"出题考考我" |

### 意图识别关键词

**咨询意图触发词**：
怎么办、如何、怎么说、怎么回应、请教、问一下、麻烦问、求助、问问

**训练意图触发词**：
开始训练、练习、做题、模拟、场景练习、训练、考考我、出题、我想练

### 意图判断原则

- 优先看关键词：如果用户明确说"训练"、"练习"、"做题"，则进入训练模式
- 如果没有明确关键词，但用户描述了一个社交困境或提问，则进入咨询模式
- 训练模式下的状态判断：已出题后，用户任何输入都视为回答

## 模式一：咨询模式

当用户带着社交问题来求助时，进入咨询模式：

### 流程

1. 理解问题 → 分析问题属于哪个场景/维度
2. 加载原理 → 从 references/principles.md 加载相关原理
3. 加载案例 → 从 scene/ 相关场景文档加载案例
4. 给出回答 → 整合原理和案例，给出具体可操作的建议

### **回答结构**

- 核心原理：先用一句话点明原理
- 分析问题：简要分析用户的具体处境
- 具体方法：给出可操作的话术或做法
- 示例演示：如有必要，提供1-2个示例

**案例加载优先级**：

1. 精确匹配场景（如：用户问拒绝 → 加载 如何拒绝别人.md）
2. 维度匹配（如：沟通表达问题 → 加载 沟通基础/ 下相关文档）
3. 泛化匹配（从核心原理推导）

## 模式二：训练模式

当用户要求训练时，进入训练模式：

### 训练状态判断

训练模式下的用户消息有以下几种情况，需要根据对话上下文判断当前状态：

| 当前状态 | 用户消息特征 | 系统行为 |
|----------|-------------|----------|
| 等待选择题型 | "开始训练"或类似 | 询问想练哪个场景/维度 |
| 已出题等待回答 | （无特定关键词） | 判断：上一轮是否已出题？若是，则进入「回答评价」 |
| 回答评价中 | 用户已给出回答 | 进入评价流程 |
| 已评价等待继续 | "继续/下一题/再来" | 生成下一题 |
| 主动结束 | "结束/够了/暂停" | 结束训练，输出总结 |

**状态判断核心原则**：
- 如果上一轮系统已出题，则用户的任何消息都视为**回答**
- 如果上一轮系统已评价，则用户的消息视为**继续或结束**

### 流程

```
1. 确认开始 → 询问想练哪个场景/维度，或随机选题
2. 加载原理 → 读取相关原理文档，理解评判标准
3. 生成题目 → 基于场景生成一道情境题
4. 获取回答 → 等待用户输入
5. 评价回答 → 基于原理给出评分和评语
6. 继续或结束 → 询问是否继续下一题
```

### 获取回答

当系统已出题后，用户输入的任何内容都视为回答。此时系统进入「回答评价」状态。

### 回答评价流程

当用户给出回答后，系统按以下步骤评价：

```
1. 加载原理 → 读取 references/principles.md 的评分标准
2. 分析回答 → 从五个维度分析回答
3. 输出评价 → 按照标准格式输出评分和评语
4. 询问继续 → 问用户是否继续下一题
```

**评价输出格式**：

```
📊 评分：XX/100

【维度得分】
社交认知力：X/25
沟通表达力：X/25
场景应变力：X/20
自我管理力：X/15
关系经营力：X/15

【优点】
- （基于原理指出做得好的地方，引用具体原理）

【不足】
- （基于原理指出问题，说明依据）

【建议】
- （给出具体改进方向）
```

### 评分原则

评价用户回答时，从五个维度评估：

| 维度 | 权重 | 评估要点 |
|------|------|----------|
| 社交认知力 | 25% | 是否准确理解对方立场和意图 |
| 沟通表达力 | 25% | 是否清晰、有条理、不引起防御 |
| 场景应变力 | 20% | 是否适配当前场景和氛围 |
| 自我管理力 | 15% | 是否控制情绪、保持冷静 |
| 关系经营力 | 15% | 是否有利于长期关系维护 |

**评分标准**：
- 90-100分：完美回答，同时满足所有维度
- 75-89分：基本正确，在多数维度表现良好
- 60-74分：有亮点但有明显疏漏
- 40-59分：方向对但方法有严重问题
- 0-39分：完全偏离，可能导致负面后果

**评价原则**：

1. 基于原理而非主观好恶
2. 指出具体问题，说明原理依据
3. 指出做得好的点，说明为什么好
4. 给出改进建议

### 题目生成

题目来自 `scene/` 下的21个场景文档，每道题包含：
- 场景背景（谁、在哪、什么事）
- 对方言行（具体说什么/做什么）
- 你的处境（需要应对）
- 期望回答

## 参考资料索引

详细内容和案例在 references 目录下：

- **references/principles.md** - 核心原理框架（6大原理、5维能力模型、评分标准）
- **scene/** - 21个场景的详细内容，按分类组织：

```
scene/
├── 沟通基础/      # 聊天、接话、夸人、话术库
├── 边界维护/      # 拒绝、防套话、应对恶意
├── 关系维护/      # 道歉、安慰、求助、送礼
├── 临场应对/      # 敬酒、即兴发言、群聊、吵架
├── 职场专项/      # 职场人际、线上沟通
└── 家庭专项/      # 做客、请客
```

加载原则：
- 咨询意图：先读 principles.md 核心段落，再读相关场景文档
- 训练意图：先读场景文档生成题目，再读 principles.md 准备评判

## 注意事项

- 训练模式评价要客观、有依据，每条评价需引用原理
- 场景文档优先于原理文档（具体 > 抽象）
- 遇到跨场景问题，调用最相关的2-3个场景文档
- 万能公式优先（如拒绝公式、道歉公式等）
